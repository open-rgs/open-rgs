# ADR 005  - RGS generates round IDs

**Status:** Accepted
**Date:** 2026-05-08

## Context

Every round needs a unique ID for audit, idempotency, and client-side
correlation. Three candidates for who generates it:

- **RGS** generates; adapter forwards.
- **Wallet** generates; adapter returns to RGS.
- **Adapter** generates; RGS uses.

## Decision

**RGS generates the round ID.** Adapter forwards it to the wallet on
the relevant RPC. If the wallet's API insists on a wallet-generated ID,
the adapter is responsible for either (a) overwriting on response or
(b) refusing the integration.

The ID is generated using the configured idempotency strategy
(`createServer({ idempotency: { generate: uuidV4 | cuidV2 | custom } })`).

## Consequences

**Upsides:**

- RGS knows the round ID at the moment it logs the round-start event.
  No "round started but ID is pending" weirdness in our logs.
- Idempotency keys generated by RGS can serve double duty as round
  IDs, simplifying the retry story.
- Same generation strategy across providers  - one config knob
  controls round-ID flavor.
- Wallet RPCs can be retried safely with the same ID, because the
  ID is stable on our side from the moment the round starts.

**Costs:**

- Adapters integrating with wallets that generate their own round
  ID need to translate: send the RGS-generated ID, capture the
  wallet-generated ID, store the mapping. (Most wallet APIs accept
  client-generated IDs; this is the rare case.)
- The "configurable generator" option means we have to ship at
  least two implementations (uuid-v4 default, cuid-v2 alternative).
  Mitigated by: they're 30 lines each.

## Alternatives considered

- **Wallet generates.** RGS would have to round-trip and wait before
  it can emit a "round started" log entry, and idempotency keys
  would diverge from round IDs.
- **Both generate, RGS uses its own internally and wallet's
  externally.** Maps two IDs through every adapter call. Rejected
  for complexity.
- **Random opaque server-side ID, no idempotency role.** Loses the
  "key serves double duty" simplification.
